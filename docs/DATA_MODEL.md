# Data Model

BarBacker uses Cloud Firestore (NoSQL).

## Collections

### `bars` (Collection)
Represents a physical bar location.
- `id` (string): Unique ID (often generated).
- `name` (string): Name of the bar.
- `location` (GeoPoint): Lat/Lng coordinates.
- `address` (string): Human-readable address.
- `menu` (Array<Object>): Configurable buttons for the bar.
  - `label` (string): Text on button (e.g., "ICE").
  - `icon` (string): Icon identifier.
  - `category` (string): Grouping.

### `requests` (Collection)
Represents a task/request generated by a Bartender.
- `id` (string): Unique ID.
- `barId` (string): ID of the bar this request belongs to.
- `type` (string): Request type (e.g., "ICE", "GLASSWARE").
- `status` (string): "open", "claimed", "completed".
- `timestamp` (Timestamp): When created.
- `createdBy` (string): User ID of requester.
- `claimedBy` (string): User ID of claimer (optional).

### `users` (Collection)
Represents a staff member.
- `uid` (string): Firebase Auth UID.
- `name` (string): Display name.
- `barId` (string): Current bar they are working at.
- `role` (string): "bartender", "barback".
- `status` (string): "active", "pending", "off_clock".
- `fcmToken` (string): Push notification token.

### `notices` (Collection)
Global or bar-specific messages/alerts.
- `text` (string): Content.
- `barId` (string): Target bar.
- `timestamp` (Timestamp): Created time.
